# fluffy-octo-carnival

Алгоритм решения задачи о дробном рюкзаке
Постановка проблемы
Рассмотрим задачу оптимального выбора объектов для заполнения ограниченной емкости. Имеется множество предметов, каждый из которых обладает определенным весом и стоимостью. Также задана максимальная грузоподъемность контейнера. Необходимо определить такой набор предметов или их частей, который обеспечит максимальную суммарную стоимость при условии соблюдения ограничения по весу.

Принцип работы алгоритма
В основе метода лежит стратегия последовательного выбора наиболее эффективных объектов. Алгоритм состоит из трех основных этапов:

Первым шагом вычисляется показатель эффективности для каждого предмета - отношение его стоимости к весу. Этот коэффициент показывает, насколько ценным является единица массы объекта.

На втором этапе производится упорядочивание всех предметов по убыванию рассчитанного коэффициента эффективности. Таким образом, в начале списка оказываются наиболее "выгодные" объекты.

Третий этап представляет собой процесс заполнения контейнера. Мы последовательно перебираем предметы из отсортированного списка и помещаем их в рюкзак. Если текущий предмет полностью помещается в оставшееся пространство, мы берем его целиком. В противном случае мы берем только ту часть предмета, которая может поместиться в оставшееся пространство.

Анализ эффективности алгоритма
С точки зрения вычислительной сложности, наиболее затратной операцией является сортировка предметов. Временная сложность этой операции составляет O(n log n), где n - количество предметов. Сам процесс отбора предметов после сортировки выполняется за линейное время O(n).

Важным преимуществом данного подхода является то, что для задачи дробного рюкзака он обеспечивает абсолютно оптимальное решение. Это означает, что полученный результат невозможно улучшить - он всегда будет иметь максимально возможную стоимость. Коэффициент аппроксимации равен единице, что подтверждает оптимальность метода.

Контрольный вопрос 3: Схемы приближенного решения сложных задач
PTAS и FPTAS представляют собой специализированные подходы для нахождения приближенных решений computationally сложных задач, которые не могут быть эффективно решены точно за полиномиальное время.

PTAS (Polynomial Time Approximation Scheme) - это схема приближения, которая позволяет найти решение, сколь угодно близкое к оптимальному. Точность решения регулируется параметром ε. Время работы PTAS полиномиально зависит от размера входных данных, однако может экспоненциально возрастать при стремлении к высокой точности (при уменьшении ε). Это означает, что для очень малых значений ε алгоритм может стать практически неприменимым.

FPTAS (Fully Polynomial Time Approximation Scheme) является более строгим вариантом PTAS. В этом случае время работы алгоритма полиномиально зависит как от размера задачи, так и от величины 1/ε. Такая характеристика делает FPTAS значительно более практичным инструментом, поскольку он сохраняет эффективность даже при требовании очень высокой точности решения.

Основное различие между этими подходами заключается в том, что PTAS обеспечивает полиномиальную зависимость только от размера задачи, в то время как FPTAS гарантирует полиномиальную зависимость и от размера задачи, и от требуемой точности. Наличие FPTAS для конкретной задачи считается особенно ценным с практической точки зрения, поскольку позволяет гибко управлять точностью решения без катастрофического роста вычислительных затрат.
